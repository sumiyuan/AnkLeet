**background.js -- modify GET_AI_FEEDBACK handler (around line 138-145):**

After the existing `sendResponse({ feedback })` line inside the GET_AI_FEEDBACK handler, add hint seeding logic. This runs AFTER sendResponse so it does not block the wrong-submission panel from receiving its response:

1. Load or create the conversation for `submission.titleSlug`:
   ```
   let conversation = await getConversation(db, submission.titleSlug);
   const now = Date.now();
   if (!conversation) {
     conversation = { titleSlug: submission.titleSlug, messages: [], createdAt: now, updatedAt: now };
     conversation.messages.push(buildSystemPrompt(submission.titleSlug));
   }
   ```

2. Append a user message describing what happened (so the AI has context):
   ```
   const modeLabel = message.payload.mode === 'hint' ? 'hint' : 'full solution';
   conversation.messages.push({
     role: 'user',
     content: `I submitted a wrong answer and asked for a ${modeLabel}.`,
     timestamp: now
   });
   ```

3. Append the AI feedback as an assistant message:
   ```
   conversation.messages.push({ role: 'assistant', content: feedback, timestamp: now });
   conversation.updatedAt = now;
   ```

4. Save: `await putConversation(db, conversation);`

5. Signal content-chat.js to reload (fire-and-forget):
   ```
   try {
     await chrome.tabs.sendMessage(sender.tab.id, {
       type: 'SHOW_CHAT_SEED',
       titleSlug: submission.titleSlug
     });
   } catch { /* tab navigated away */ }
   ```

**Important:** The seeding logic must be INSIDE the try block that already wraps the callOpenRouter call, AFTER `sendResponse({ feedback })`. The `sendResponse` call sends the response to content-toast.js. The seeding runs as a follow-up in the same async IIFE. Use `submission.titleSlug` (available in handler scope from `getSubmissionById` result). Use `sender.tab.id` for the tab message.

**content-chat.js -- add SHOW_CHAT_SEED listener:**

Add a `chrome.runtime.onMessage.addListener` at the module level of content-chat.js:

```javascript
chrome.runtime.onMessage.addListener(function(msg) {
  if (msg.type === 'SHOW_CHAT_SEED') {
    reloadConversation(msg.titleSlug);
  }
});
```

The `reloadConversation` function (created in Plan 01) calls CHAT_LOAD_CONVERSATION and re-renders messages. If the panel is not open, the seeded message will appear the next time the user opens it.

**Do NOT modify content-toast.js** (per STATE.md decision).
