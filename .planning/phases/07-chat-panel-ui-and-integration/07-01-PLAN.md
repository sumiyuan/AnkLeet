  <name>Task 1: Create content-chat.js with chat button, panel, messaging, markdown renderer, and SPA navigation</name>
  <files>extension/content-chat.js, extension/manifest.json</files>
  <action>
Create `extension/content-chat.js` as a new Shadow DOM content script following the exact patterns from `content-toast.js`. This single file implements the entire chat panel UI.

**Chat button (CHAT-01):**
- Create a fixed-position floating action button (bottom-right corner, above any existing toast elements) with a chat icon (use a simple SVG speech bubble or Unicode character)
- Button has `position: fixed; bottom: 24px; right: 24px; z-index: 2147483647; width: 48px; height: 48px; border-radius: 50%; background: #FF6B00; color: white; cursor: pointer; box-shadow: 0 2px 8px rgba(0,0,0,0.3)`
- On click, toggle the chat panel visibility

**SPA navigation detection (CHAT-01):**
- Use MutationObserver on `document.body` with `{ childList: true, subtree: true }` to detect pathname changes
- Track `lastPath = location.pathname`; on change, if still on `/problems/*`, call `reinitChatPanel(newSlug)` which removes old host and re-injects for new problem
- If navigated away from `/problems/*`, remove the host element entirely
- Extract titleSlug via `location.pathname.split('/')[2]`

**Panel structure (CHAT-02):**
- Shadow DOM host: `div#leetreminder-chat-host` appended to `document.body`, `attachShadow({ mode: 'closed' })`
- Panel: fixed position, right side, `width: 380px; height: 520px; bottom: 80px; right: 24px; border-radius: 12px; background: #1a1a2e; color: #e0e0e0; font-family: -apple-system, BlinkMacSystemFont, sans-serif; box-shadow: 0 8px 32px rgba(0,0,0,0.4); display: none` (hidden by default)
- Use dark theme colors to match LeetCode's dark mode aesthetic
- Header: "AI Chat" title, problem slug display (formatted: replace hyphens with spaces, title case), "New Chat" button, close "x" button
- Messages area: scrollable container (`overflow-y: auto; flex: 1`), displays user and assistant message bubbles
- User messages: right-aligned, orange background (#FF6B00), white text, rounded corners
- Assistant messages: left-aligned, dark gray background (#2d2d44), light text, rounded corners
- Input area: textarea (fixed height 60px, resize: none, dark background #16162a, light text, border: 1px solid #3d3d5c) + Send button (orange background)
- Loading indicator: "Thinking..." text with simple CSS animation (opacity pulse), hidden by default
- Error message area: red text, hidden by default

**Keyboard handling (from research pitfalls):**
- Textarea keydown: `event.stopPropagation()` on ALL key events to prevent LeetCode's Monaco editor from intercepting
- Enter (without Shift): `event.preventDefault()`, trigger send
- Shift+Enter: allow newline (default behavior)

**Send message flow (CHAT-02, CHAT-05):**
- On send: show loading indicator, disable send button, append user message bubble to messages area
- Call `chrome.runtime.sendMessage({ type: 'CHAT_SEND_MESSAGE', payload: { titleSlug, content } }, callback)`
- On success: hide loading, append assistant message bubble (rendered with markdown), scroll to bottom
- On error: hide loading, show error message inline (red text below messages area), re-enable send button
- Handle `chrome.runtime.lastError` and missing response

**Load conversation on panel open:**
- When panel is shown, call `CHAT_LOAD_CONVERSATION` with current titleSlug
- If conversation exists, render all non-system messages (skip `role: 'system'`) as bubbles in the messages area
- If no conversation, show empty state

**New Chat (CONV-02):**
- "New Chat" button in header calls `CHAT_CLEAR_CONVERSATION` with current titleSlug
- On success: clear all message bubbles from messages area, clear error message
- On error: show error message

**Markdown renderer (CHAT-04):**
- `renderMarkdown(container, text)` function that processes AI response text into DOM elements
- Handle triple-backtick code fences: extract language hint, create `pre > code` elements with `textContent` (never innerHTML)
- Handle `**bold**` text: create `strong` elements
- Handle `` `inline code` ``: create `code` elements with monospace background
- Handle bullet lists (`- ` or `* ` prefixed lines): create `ul > li` elements
- Handle numbered lists (`1. ` prefixed lines): create `ol > li` elements
- Handle regular paragraphs: create `p` elements
- ALL text content set via `textContent` or `createTextNode` -- NEVER innerHTML for user or AI sourced text
- Style code blocks: `background: #0d0d1a; padding: 12px; border-radius: 6px; overflow-x: auto; font-family: monospace; font-size: 13px; white-space: pre`
- Style inline code: `background: #0d0d1a; padding: 2px 6px; border-radius: 3px; font-family: monospace; font-size: 13px`

**Auto-scroll:**
- After appending any new message, set `messagesArea.scrollTop = messagesArea.scrollHeight`

**manifest.json update:**
- Add a new entry to the `content_scripts` array:
  ```json
  {
    "matches": ["https://leetcode.com/problems/*"],
    "js": ["content-chat.js"],
    "run_at": "document_end"
  }
  ```

**CSS reset on host:**
- Apply `all: initial` on the outermost shadow DOM element to prevent LeetCode style inheritance
- All styles scoped inside shadow root `<style>` element

**Important constraints:**
- Follow content-toast.js patterns exactly: closed shadow root, createElement for all DOM, textContent for all external text
- Do NOT use innerHTML for any user-sourced or AI-sourced content
- Do NOT use chrome.sidePanel API (per STATE.md decision)
- Do NOT modify content-toast.js (per STATE.md decision)
