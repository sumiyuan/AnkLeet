<task type="auto">
  <name>Task 1: Vendor ts-fsrs and migrate IndexedDB to version 2</name>
  <files>extension/lib/ts-fsrs.umd.js, extension/background.js</files>
  <action>
1. Create `extension/lib/` directory and download the ts-fsrs UMD bundle:
   ```bash
   mkdir -p extension/lib
   curl -o extension/lib/ts-fsrs.umd.js https://cdn.jsdelivr.net/npm/ts-fsrs@5.2.3/dist/index.umd.js
   ```

2. Add `importScripts('lib/ts-fsrs.umd.js');` at the top of background.js (before any other code). After import, verify the UMD global name by inspecting the downloaded file header — research suggests `self.tsfsrs` but this must be confirmed from the actual file. Extract `createEmptyCard`, `fsrs`, `Rating`, `State` from whatever global the UMD exposes.

3. Update `openDatabase()` to open version 2 instead of 1. Extend the `onupgradeneeded` handler:
   - Wrap existing submissions store creation in `if (event.oldVersion < 1)` guard
   - Add `if (event.oldVersion < 2)` block that creates:
     - `cards` store with keyPath `titleSlug`, indexes: `due` (non-unique), `state` (non-unique)
     - `reviewLogs` store with keyPath `id` (autoIncrement), indexes: `titleSlug` (non-unique), `reviewedAt` (non-unique)
   - See 02-RESEARCH.md "Pattern 1" for exact schema

4. Add version change handlers for multi-tab safety:
   - `request.onblocked = () => {}` (silently wait)
   - After `request.onsuccess`, add `db.onversionchange = () => db.close()` so existing connections yield on future upgrades

5. Add two helper functions for card store access:
   - `getCard(database, titleSlug)` — reads a single card by keyPath from `cards` store. Returns the card object or null.
   - `putCard(database, card)` — writes (upserts) a card to `cards` store using `store.put(card)`.
   - `addReviewLog(database, logEntry)` — writes a review log entry to `reviewLogs` store using `store.add(logEntry)`.
  </action>
  <verify>
    <automated>ls -la extension/lib/ts-fsrs.umd.js && head -5 extension/lib/ts-fsrs.umd.js && grep -c "oldVersion < 2" extension/background.js && grep -c "importScripts" extension/background.js</automated>
  </verify>
  <done>ts-fsrs.umd.js exists in extension/lib/ (~72KB), background.js imports it via importScripts, openDatabase opens version 2 with cards and reviewLogs stores created in onupgradeneeded, helper functions getCard/putCard/addReviewLog exist</done>
</task>

<task type="auto">
  <name>Task 2: Create FSRS card on first Accepted submission</name>
  <files>extension/background.js</files>
  <action>
1. Create `maybeCreateCard(database, titleSlug)` function:
   - Call `getCard(database, titleSlug)` — if card exists, return early (idempotent)
   - Call `createEmptyCard(new Date())` from ts-fsrs to get initial card state
   - Build card object matching the schema from 02-RESEARCH.md:
     ```
     { titleSlug, due (ISO string), stability, difficulty, elapsed_days, scheduled_days, reps, lapses, state, last_review (null), createdAt (Date.now()) }
     ```
   - Store dates as ISO strings (not Date objects) per research guidance
   - Write using `store.add(card)` with ConstraintError handling for race conditions (same pattern as submissions)

2. Modify `saveSubmission()` to call `maybeCreateCard` after a successful `addRecord`:
   - Only call when the submission is Accepted: check `record.statusDisplay === 'Accepted'`
   - Call `maybeCreateCard(db, record.titleSlug)` — do not await blocking; use `.catch()` for error logging
   - The card creation is fire-and-forget relative to the submission save — submission capture must not fail if card creation fails

3. Ensure the `if (!db)` re-open guard covers `maybeCreateCard` (it already covers `saveSubmission` which calls it, so this is inherited).
  </action>
  <verify>
    <automated>grep -c "maybeCreateCard" extension/background.js && grep "statusDisplay.*Accepted" extension/background.js && grep "createEmptyCard" extension/background.js</automated>
  </verify>
  <done>maybeCreateCard function exists, saveSubmission calls it for Accepted submissions only, card stored with ISO string dates and all FSRS fields, duplicate cards prevented via ConstraintError guard</done>
</task>
