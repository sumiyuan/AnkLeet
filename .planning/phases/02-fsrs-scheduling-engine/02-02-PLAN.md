<task type="auto">
  <name>Task 1: Implement RATE_REVIEW and GET_DUE_TODAY message handlers</name>
  <files>extension/background.js</files>
  <action>
1. Create `rateReview(database, titleSlug, ratingName)` async function:
   - Read card via `getCard(database, titleSlug)`. Throw if not found.
   - Reconstruct Date fields before passing to scheduler (CRITICAL per research pitfall):
     ```javascript
     const card = { ...stored, due: new Date(stored.due), last_review: stored.last_review ? new Date(stored.last_review) : null };
     ```
   - Create scheduler: `const scheduler = fsrs();` (default params, request_retention=0.9)
   - Call `scheduler.repeat(card, new Date())` to get recordLog with all four rating outcomes
   - Map rating name to ts-fsrs Rating enum: `Rating[ratingName]` (e.g., Rating.Good = 3, Rating.Again = 1). Validate ratingName is one of Again/Hard/Good/Easy.
   - Extract `{ card: newCard, log }` from `recordLog[rating]`
   - Build review log entry:
     ```javascript
     { titleSlug, rating: log.rating, oldState: stored.state, newState: newCard.state, scheduledDays: log.scheduled_days, elapsedDays: log.elapsed_days, reviewedAt: new Date().toISOString() }
     ```
   - Build updated card object (same shape as creation, all dates as ISO strings):
     ```javascript
     { titleSlug, due: newCard.due.toISOString(), stability: newCard.stability, difficulty: newCard.difficulty, elapsed_days: newCard.elapsed_days, scheduled_days: newCard.scheduled_days, reps: newCard.reps, lapses: newCard.lapses, state: newCard.state, last_review: newCard.last_review?.toISOString() ?? null, createdAt: stored.createdAt }
     ```
   - Call `putCard(database, updatedCard)` then `addReviewLog(database, reviewLogEntry)`

2. Create `getDueToday(database)` async function:
   - Calculate end of today: `const end = new Date(); end.setHours(23, 59, 59, 999);`
   - Use `IDBKeyRange.upperBound(end.toISOString(), false)` on the `due` index of `cards` store
   - Return all matching cards via `index.getAll(range)`

3. Update the `chrome.runtime.onMessage.addListener` handler to process new message types:
   - IMPORTANT: The handler must `return true` for async message types (RATE_REVIEW, GET_DUE_TODAY, GET_STATS) to signal Chrome that sendResponse will be called asynchronously
   - Add `if (!db)` re-open guard before processing any of the new message types (same pattern as saveSubmission)
   - RATE_REVIEW: call `rateReview(db, payload.titleSlug, payload.rating)`, sendResponse({ ok: true }) on success, sendResponse({ error: message }) on failure
   - GET_DUE_TODAY: call `getDueToday(db)`, sendResponse({ cards: result })
   - Keep existing SUBMISSION_CAPTURED handling unchanged
  </action>

<task type="auto">
  <name>Task 2: Implement GET_STATS message handler with retention, count, and streak</name>
  <files>extension/background.js</files>
  <action>
1. Create `getAllReviewLogs(database)` helper:
   - Read all entries from `reviewLogs` store via `store.getAll()`
   - Return the array of log entries

2. Create `computeStreak(reviewDays)` helper:
   - Input: Set of date strings in 'YYYY-MM-DD' format (days that had at least one review)
   - Start from today, walk backward day by day
   - Count consecutive days that appear in the set
   - Stop when a day is missing from the set
   - Return the count
   - Note: This is the "naive" streak (consecutive calendar days with reviews). The "fair" streak (skip days with nothing due) is deferred per research guidance — Phase 2 stores everything needed for Phase 3 to enhance this

3. Create `getStats(database)` async function:
   - Call `getAllReviewLogs(database)` to get all log entries
   - Compute totalReviews: `logs.length`
   - Compute retentionRate: percentage of reviews where `rating >= 3` (Good=3, Easy=4) vs total. Round to integer. Return 0 if no reviews.
   - Compute streak: extract unique 'YYYY-MM-DD' dates from `reviewedAt` field (`log.reviewedAt.slice(0, 10)`), pass to `computeStreak()`
   - Return `{ totalReviews, retentionRate, streak }`

4. Add GET_STATS to the message handler:
   - GET_STATS: call `getStats(db)`, sendResponse(result)
   - Same async pattern as the other handlers (return true, if (!db) guard)
  </action>

  </verify>
  <done>getStats returns { totalReviews, retentionRate, streak }. retentionRate = percentage of Good/Easy ratings. streak = consecutive calendar days with at least one review (counting backward from today). GET_STATS message type handled in onMessage listener.</done>
