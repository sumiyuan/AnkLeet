<task type="auto">
  <name>Task 2: Implement service worker with IndexedDB storage layer</name>
  <files>extension/background.js</files>
  <action>
Create `extension/background.js` — the MV3 service worker. ALL event listeners MUST be registered at the top level (global scope), never inside callbacks or async functions.

**Structure (follow Pattern 3 + Pattern 4 from research):**

1. **Top-level IndexedDB connection** — call `openDatabase()` eagerly; store result in module-scope `let db = null;`

2. **`openDatabase()` function** — opens IndexedDB database named `'leetreminder'` at version 1. In `onupgradeneeded`:
   - Create object store `'submissions'` with `keyPath: 'id'` and `autoIncrement: true`
   - Create unique index `'submissionId'` on `'submissionId'` field
   - Create non-unique index `'titleSlug'` on `'titleSlug'` field
   - Create non-unique index `'capturedAt'` on `'capturedAt'` field

3. **Top-level `chrome.runtime.onMessage.addListener`** — handles message types:
   - `'SUBMISSION_CAPTURED'`: calls `saveSubmission(message.payload, sender.tab.id)`, returns `false` (no async response)

4. **Top-level `chrome.runtime.onInstalled.addListener`** — on install, write default settings to `chrome.storage.local`:
   ```javascript
   chrome.storage.local.set({ settings: { captureEnabled: true } });
   ```

5. **`saveSubmission(data, tabId)` async function** — transforms the raw `submissionDetails` GraphQL payload into the storage record shape:
   ```javascript
   {
     submissionId: String(data.id || data.submissionId),
     titleSlug: data.question?.titleSlug,
     title: data.question?.title,
     difficulty: data.question?.difficulty,
     topicTags: (data.question?.topicTags || []).map(t => t.name),
     url: `https://leetcode.com/problems/${data.question?.titleSlug}/`,
     code: data.code,
     lang: data.lang?.name,
     langDisplay: data.lang?.verboseName,
     statusDisplay: data.statusDisplay,
     capturedAt: Date.now()
   }
   ```
   Uses `store.add(record)` — NOT `put` — so the unique index on `submissionId` prevents duplicates. On `ConstraintError`, silently skip (duplicate submission). On success, send `{ type: 'SHOW_TOAST' }` to the source tab via `chrome.tabs.sendMessage(tabId, ...)`, wrapped in try/catch.

6. **`notifyTab(tabId)` async function** — sends `{ type: 'SHOW_TOAST' }` to the tab, wrapped in try/catch (tab may have navigated away).

**Error handling strategy (Claude's discretion):** Silent fail + console.warn for unexpected data shapes. Log `console.warn('[LeetReminder] Unexpected submissionDetails shape', data)` when required fields (submissionId, question) are missing. Do NOT show user-facing errors in Phase 1 — the toast only fires on successful saves.

**Important:** If `db` is null when `saveSubmission` is called (worker just restarted), call `openDatabase()` again to get a fresh connection before proceeding.
