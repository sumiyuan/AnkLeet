Add alarm, badge, and notification logic to the existing background.js. ALL new event listeners MUST be registered at the top level (module scope), NOT inside async callbacks.

1. **updateBadge(count) function** — Add near the other utility functions:
   - If count > 0: `chrome.action.setBadgeText({ text: String(count) })` and `chrome.action.setBadgeBackgroundColor({ color: '#E05C5C' })`
   - If count === 0: `chrome.action.setBadgeText({ text: '' })`

2. **Alarm creation** — Add at top level (module scope), after the `openDatabase()` call:
   - `chrome.alarms.get('checkDueReviews', (alarm) => { if (!alarm) chrome.alarms.create('checkDueReviews', { periodInMinutes: 1 }); });`
   - This is idempotent — safe on every worker startup

3. **Immediate badge update on startup** — Add at top level, after the openDatabase().then() block:
   - Once db is ready, call getDueToday and updateBadge. Add this to the existing `.then(database => { ... })` chain: after setting db, call `getDueToday(db).then(cards => updateBadge(cards.length)).catch(() => {});`

4. **chrome.alarms.onAlarm.addListener** — Add at TOP LEVEL (same scope as onMessage.addListener):
   - CRITICAL: Must be registered synchronously at module scope. Do NOT put inside a .then() or async function.
   - When alarm.name === 'checkDueReviews':
     a. Ensure db is open (same pattern as message handlers: if (!db) try to reopen)
     b. Get due cards: `const cards = await getDueToday(db);`
     c. Update badge: `updateBadge(cards.length);`
     d. Check notification conditions:
        - Read settings: `const { settings, lastNotifiedDate } = await chrome.storage.local.get(['settings', 'lastNotifiedDate']);`
        - If `settings?.notificationsEnabled !== true` → skip notification
        - If `cards.length === 0` → skip notification
        - Check time: parse `settings.notificationTime` (default '09:00'), compare with current time. Only fire if current time >= notificationTime.
        - Check dedup: if `lastNotifiedDate === today's date string (YYYY-MM-DD)` → skip (already notified today)
        - If all conditions pass: fire notification and record lastNotifiedDate
     e. Fire notification:
        ```
        chrome.notifications.create('dueReviews', {
          type: 'basic',
          iconUrl: 'icons/icon128.png',
          title: 'LeetReminder',
          message: `You have ${count} review${count === 1 ? '' : 's'} due today.`
        });
        await chrome.storage.local.set({ lastNotifiedDate: todayStr });
        ```
        Using fixed notificationId 'dueReviews' prevents stacking — Chrome replaces existing notification with same ID.

5. **Also update badge after RATE_REVIEW** — In the existing RATE_REVIEW handler, after the successful `rateReview()` call, get updated due count and call updateBadge. This ensures the badge is current after rating from the popup.

ANTI-PATTERNS TO AVOID:
- Do NOT use setInterval — it dies when worker terminates
- Do NOT register onAlarm listener inside a callback — must be top-level
- Do NOT fire notification on every alarm tick — use lastNotifiedDate dedup
- Do NOT overwrite settings object when writing lastNotifiedDate — use separate key
