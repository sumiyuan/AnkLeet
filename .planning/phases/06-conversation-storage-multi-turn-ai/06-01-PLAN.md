<task type="auto">
  <name>Task 2: Add CHAT_SEND_MESSAGE, CHAT_LOAD_CONVERSATION, and CHAT_CLEAR_CONVERSATION handlers</name>
  <files>extension/background.js</files>
  <action>
Add three new message handler blocks inside the existing `chrome.runtime.onMessage.addListener` callback, after the `GET_AI_FEEDBACK` handler and before the final `return false;`.

**1. CHAT_SEND_MESSAGE handler:**
- Expects `message.payload = { titleSlug: string, content: string }`
- Follow the async IIFE + `return true` pattern from existing handlers
- Open DB if needed (same guard pattern)
- Read API key and model from `chrome.storage.local.get('settings')` -- if no API key, sendResponse with error
- Load existing conversation via `getConversation(db, titleSlug)` or create new: `{ titleSlug, messages: [], createdAt: Date.now(), updatedAt: Date.now() }`
- If conversation.messages is empty (new conversation), prepend system prompt via `buildSystemPrompt(titleSlug)` -- push it to messages array
- Append user message: `{ role: 'user', content, timestamp: Date.now() }`
- Update `conversation.updatedAt = Date.now()`
- Cap messages for API: `const messagesToSend = conversation.messages.slice(-10)` -- strip timestamps before sending (OpenRouter expects only role+content): `messagesToSend.map(m => ({ role: m.role, content: m.content }))`
- Set up keepalive interval: `const keepAlive = setInterval(() => chrome.storage.local.get('_ping'), 20_000)` (same pattern as GET_AI_FEEDBACK)
- Call `callOpenRouter(apiKey, model, strippedMessages)` in try block
- On success: append assistant message `{ role: 'assistant', content: reply, timestamp: Date.now() }`, update `conversation.updatedAt`, call `putConversation(db, conversation)`, sendResponse `{ ok: true, reply, messages: conversation.messages }`
- On error: sendResponse `{ error: err.message }`
- In finally: `clearInterval(keepAlive)`

**2. CHAT_LOAD_CONVERSATION handler:**
- Expects `message.payload = { titleSlug: string }`
- Async IIFE + return true pattern
- Open DB if needed
- Call `getConversation(db, message.payload.titleSlug)`
- sendResponse `{ conversation: conversation || null }`
- On error: sendResponse `{ error: err.message }`

**3. CHAT_CLEAR_CONVERSATION handler:**
- Expects `message.payload = { titleSlug: string }`
- Async IIFE + return true pattern
- Open DB if needed
- Call `deleteConversation(db, message.payload.titleSlug)`
- sendResponse `{ ok: true }`
- On error: sendResponse `{ error: err.message }`

IMPORTANT: The system prompt message stored in conversation.messages has no timestamp field (just role and content). User and assistant messages have timestamp fields. When building messagesToSend for the API, strip the timestamp field from ALL messages since OpenRouter only accepts {role, content}.
