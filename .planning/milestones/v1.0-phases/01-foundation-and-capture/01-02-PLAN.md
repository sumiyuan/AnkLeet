  <files>extension/content-main.js, extension/content-isolated.js</files>
  <action>
**content-main.js** (MAIN world, document_start) — Override `window.fetch` before LeetCode's scripts load. Follow Pattern 1 from research.

1. Store reference to original fetch: `const originalFetch = window.fetch;`
2. Override `window.fetch = async function(...args) { ... }`
3. Inside override:
   - Call `const response = await originalFetch.apply(this, args);`
   - Extract URL from args: `const url = typeof args[0] === 'string' ? args[0] : args[0]?.url;`
   - Check if URL matches LeetCode GraphQL: `url && url.includes('/graphql')`
   - If GraphQL match:
     - Also check the request body for `operationName: "submissionDetails"` — read from `args[1]?.body` if it's a string, parse as JSON, check `operationName`. This prevents intercepting ALL GraphQL calls (LeetCode makes many).
     - Clone response: `const cloned = response.clone();`
     - Parse: `cloned.json().then(body => { ... }).catch(() => {});`
     - If `body?.data?.submissionDetails` exists, post message:
       ```javascript
       window.postMessage({
         source: 'leetreminder',
         type: 'submission',
         data: body.data.submissionDetails
       }, '*');
       ```
   - ALWAYS return `response` (the original, unconsumed response) to the page
4. Wrap the entire override in a try/catch — if anything fails, ensure `window.fetch` still works (fall through to original)

**content-isolated.js** (ISOLATED world, document_start) — Relay messages from page context to service worker. Follow Pattern 1 from research.

1. Listen for `window` message events
2. Validate: `event.source === window` AND `event.data?.source === 'leetreminder'` AND `event.data?.type === 'submission'`
3. Forward to service worker:
   ```javascript
   chrome.runtime.sendMessage({
     type: 'SUBMISSION_CAPTURED',
     payload: event.data.data
   });
   ```
4. Wrap `sendMessage` in try/catch — service worker may be restarting. On error, retry once after 500ms delay using `setTimeout`.

**Important constraints:**
- Do NOT make separate API calls to LeetCode (locked decision: parse from intercepted response only)
- Do NOT scrape the DOM for metadata
- Silent fail + console.warn on unexpected data shapes

**content-toast.js** (ISOLATED world, document_end) — Shadow DOM toast UI. Follow Pattern 5 from research.

1. **`showToast(message)` function:**
   - Remove any existing toast host first: `document.getElementById('leetreminder-toast-host')?.remove();`
   - Create host div: `const host = document.createElement('div'); host.id = 'leetreminder-toast-host';`
   - Append to body: `document.body.appendChild(host);`
   - Create closed shadow root: `const shadow = host.attachShadow({ mode: 'closed' });`
   - Create `<style>` element with toast CSS:
     - `all: initial;` to reset inherited styles
     - `position: fixed; bottom: 20px; right: 20px;` (bottom-right per locked decision)
     - `background: #1a1a1a; color: #ffffff;` dark theme
     - `padding: 10px 16px; border-radius: 6px;`
     - `font-family: system-ui, sans-serif; font-size: 14px;`
     - `z-index: 2147483647;` maximum z-index
     - `opacity: 1; transition: opacity 0.3s ease;`
     - `.toast.fade { opacity: 0; }`
   - Create toast div with class `'toast'`, set `textContent` to message
   - Append style and toast to shadow root
   - Auto-dismiss: `setTimeout` at 2000ms adds `'fade'` class, then `setTimeout` at 300ms removes host element

2. **Message listener:**
   - `chrome.runtime.onMessage.addListener((msg) => { ... });`
   - If `msg.type === 'SHOW_TOAST'`, call `showToast('✓ Submission captured')`
   - Per locked decisions: minimal content with checkmark, same appearance for accepted and wrong, auto-dismiss ~2 seconds

**Style constraints (locked decisions):**
- Same neutral appearance for both accepted and wrong submissions — no color coding by result
- Minimal content: checkmark + "Submission captured" only — no problem details
- Bottom-right corner positioning
- Auto-dismiss after ~2 seconds
