  <action>
Add three new functions and one message handler branch to content-toast.js. Insert them BEFORE the maybeBlurEditor() call (before line 350) so the message listener at the bottom can reference them. Do NOT modify any existing functions.

**1. renderFeedback(container, text)** — Minimal safe markdown renderer.
Split text on triple-backtick code fences using `text.split(/(```[\s\S]*?```)/g)`. For code fence segments: extract code body (skip language tag on first line), create `<pre>` element, set `pre.textContent = code` (XSS-safe). For text segments: create `<p>` element, set `p.textContent = seg` (XSS-safe). Clear container with `container.innerHTML = ''` before rendering. Never use innerHTML with API-sourced text.

**2. renderError(container, message)** — Create a div with class `error-msg`, set textContent to message string. Clear container first.

**3. showWrongSubmissionDialog(submissionId, titleSlug, title)** — Shadow DOM dialog following the exact showRatingDialog pattern:
- Call `removeHost()` first
- Create host div with id `leetreminder-toast-host`, attach closed shadow root
- Style block includes: `.overlay` (same as rating dialog), `.dialog` (same but `max-width: 480px` for code block readability), `.dialog-title` (red `#e05c5c` color, text "Wrong Submission"), `.dialog-problem`, `.ai-buttons` (flex row), `.ai-btn` (styled like rating-btn with `all: initial`), `.ai-btn.hint` (background `#7c6af7`), `.ai-btn.full` (background `#4caf50`), `.loading` (hidden by default, `color: #888`), `.feedback-area` (left-aligned, `max-height: 300px`, `overflow-y: auto`), `.feedback-area p` (`white-space: pre-wrap`, `line-height: 1.5`), `.feedback-area pre` (dark background `#1e1e1e`, `overflow-x: auto`, monospace, `color: #ce9178`), `.error-msg` (`color: #e05c5c`), `.skip-btn` (same as rating dialog)
- DOM structure: overlay > dialog > [titleEl, problemEl, buttonsEl(hintBtn + fullBtn), loadingEl, feedbackArea, dismissBtn]
- Problem display: `title || titleSlug.replace(/-/g, ' ')`
- Inner function `requestFeedback(mode)`: disables both buttons, shows loading text ("Getting hint..." or "Getting full solution..."), calls `chrome.runtime.sendMessage({ type: 'GET_AI_FEEDBACK', payload: { submissionId, mode } }, callback)`. Callback checks `chrome.runtime.lastError` first (re-enable buttons, show "Connection lost" error), then `!response` (re-enable, show "No response received"), then `response.error` (re-enable, show error string), then on success calls `renderFeedback(feedbackArea, response.feedback)` and keeps buttons disabled
- hintBtn click calls `requestFeedback('hint')`, fullBtn click calls `requestFeedback('full')`
- dismissBtn click removes host
- Do NOT add backdrop click dismiss (no `overlay.addEventListener('click')`) — user needs to read AI response without accidental dismissal

**4. Message handler** — Add `else if (msg.type === 'SHOW_WRONG_SUBMISSION')` branch to the existing `chrome.runtime.onMessage.addListener` at the bottom of the file. Call `showWrongSubmissionDialog(msg.submissionId, msg.titleSlug, msg.title)`.
